# Котедж на вул. Куземського 28.5 — CLAUDE.md

## Про проєкт

Це робочий простір для управління ремонтом котеджу Ігоря та Діани.
Площа: 101.34 м², 2 поверхи + горище. Дедлайн: вересень 2026.

**Тип забудови:** рядний (таунхаус) — ліва, права і задня стіни є спільними з сусідами, пробивати їх не можна. Єдина зовнішня стіна — фасад (передня). Вихід назовні для будь-яких комунікацій можливий лише через фасад або через дах.

## Планування приміщень

> Будинок вузький (~5.63 м) і глибокий (~9.75 м). Кімнати розташовані одна за одною спереду → назад. Більшість займають повну ширину. "Лівий/правий" — з боку вхідних дверей, дивлячись усередину.

### 1-й поверх (~58.87 м²)

| № | Назва | Площа | Де знаходиться |
|---|-------|-------|----------------|
| 1.1 | Студія (вітальня + кухня + обідня) | 41.17 м² | Ліво+центр по всій глибині: вітальня (передня частина, вікна на фасад) → їдальня (центр) → кухня (задня ліва частина) |
| 1.2 | Вестибюль (прихожа) | 4.00 м² | Права сторона, передня — вхід з вулиці |
| 1.3 | Санвузол | 4.16 м² | Права сторона, задній кут (поруч з кухнею; "правий верхній кут" по плану) |
| 1.4 | Комора | 3.32 м² | Під сходами (задня центральна зона) |
| 1.5 | **Хол** | — | Права сторона — від вестибюля до санвузолу: гардероб у 3 секції по стіні, у 1-й секції від вестибюля (верхня частина) — котел |

### 2-й поверх (~42.47 м²)

| № | Назва | Площа | Де знаходиться |
|---|-------|-------|----------------|
| 1.6 | Ванна | 4.63 м² | Середня зона, лівий бік — над сходами/їдальньою зоною 1-го пов. |
| 1.7 | Спальня master | 22.42 м² | **Передня зона (фасадна сторона), повна ширина ~5.63 м**; мансардна зона — скат даху безпосередньо над кімнатою |
| 1.8 | Спальня / гостьова / дитяча | 12.95 м² | **Задня зона, повна ширина ~5.63 м** (~2.3 м глибини); безпосередньо під горищем (зона піку) |
| 1.9 | Гардероб | 2.47 м² | Середня зона, правий бік (поруч з коридором) |
| — | Коридор | 6.22 м² | Середня зона, правий бік — сходовий простір; вимикач горища — права стіна коридору біля сходів |

### Горище (~13+ м²)

- Розташоване над кімнатою 1.8 + частина коридору 2-го пов.
- Доступ: люк + розкладні сходи з коридору 2-го поверху (люк у коридорній частині)
- Ширина: ~5.63 м (повна ширина будинку); глибина: ~2.3 м над 1.8 + частина коридору (⚠️ повну глибину уточнити)
- Покрівля моноухил: h≈0 у стіни спальні master (1.7) → пік над зовнішньою стіною кімнати 1.8
- Непридатна зона: частина над коридором де h < 1 м (ближче до стіни master-спальні)
- Придатна зона для зберігання: над кімнатою 1.8 (зона піку)

### Ключові просторові зв'язки

- Санвузол 1.3 (задній правий, 1-й пов.) → над ним: спальня 1.8 (задня зона 2-го пов.)
- Кухня (задня ліва зона, 1-й пов.) → над нею: спальня 1.8 (задня зона 2-го пов.)
- Їдальня (середня ліва зона, 1-й пов.) → над нею: ванна 1.6 (ліво, середня зона 2-го пов.)
- Ванна 1.6 (2-й пов.) → горище безпосередньо над нею → вертикальний вивід на дах (найкоротший маршрут)
- Санвузол 1.3 (1-й пов.) → вентиляція: горизонтально на фасад АБО вертикально через перекриття і 2-й пов. на дах
- Дві зони утеплення: **мансарда** (над 1.7 — утеплюємо скати) і **холодне горище** (над 1.8 — утеплюємо перекриття)

### Схема (вигляд зверху, фасад знизу)

```
ЗАДНЯ СТІНА (глуха, спільна з сусідом)
┌───────────────────────────────────────────┐
│          1-й поверх                       │
│  кухня (студія 1.1) │  1.3 Санвузол       │
│  їдальня (студія)   │  1.5 Хол            │
│                     │  сходи+1.4 Комора   │
│  вітальня (студія)  │  1.5 Хол (гардероб) │
│                     │  1.2 Вестибюль      │
└───────────────────────────────────────────┘
ФАСАД — вхід

┌───────────────────────────────────────────┐
│          2-й поверх                       │
│       1.8 Спальня (повна ширина)          │  ← задня зона
├──────────────────────┬────────────────────┤
│  1.6 Ванна (ліво)    │ Коридор+1.9 (право)│  ← середня зона
├──────────────────────┴────────────────────┤
│     1.7 Спальня master (повна ширина)     │  ← передня зона
└───────────────────────────────────────────┘
ФАСАД

Горище: над 1.8 + коридорна частина; пік над 1.8, h≈0 біля стіни master
```

## Як поводитися

- Мова спілкування: **українська**
- **Ніколи не вигадуй розміри, відстані, площі чи кількості** — якщо конкретне значення не зазначено в робочих файлах або повідомленні Ігоря, явно запитай або скажи «не вказано»
- Коли Ігор повідомляє про завершену роботу → оновлюй чекбокси в `TASKS.md` та статус в `STATUS.md`
- Коли Ігор повідомляє про витрату → оновлюй `BUDGET.md`
- Коли приймається нове технічне рішення → додавай в `SPECS.md` або `ROOMS.md` **і в `MYKOLA.md`** (секція «Погоджені технічні рішення»)
- Коли приймається рішення з таблиці «Відкриті рішення» в `STAGES.md` → оновлюй статус у тій таблиці та знімай блокер з відповідного етапу
- Коли завершується етап або починається новий → оновлюй статус у `STAGES.md` (загальна схема та секція відповідного етапу)
- Перед будь-якою відповіддю — читай відповідний файл, щоб бути в актуальному контексті; якщо питання стосується поточних робіт або інструкцій для прораба — читай також `MYKOLA.md`; якщо питання стосується планування або послідовності кроків — читай також `STAGES.md`
- Не дублюй інформацію між файлами без потреби
- Дати вказуй у форматі ДД.ММ.РРРР

## Стиль написання в MYKOLA.md

Всі секції «Погоджені технічні рішення» пишуться в єдиному стилі:

- **Тільки плоский список** — жодних вкладених підпунктів
- **Формат рядка:** `- Параметр: **значення** (примітка)` — для специфікацій
- **Формат рядка:** `- **Ключове правило** пояснення` — для обмежень і вимог
- **Bold** — лише для ключових значень, розмірів і правил; не для всього тексту
- Em dash (—) для пояснень і уточнень
- ⚠️ — тільки для критичних попереджень
- **Без абзаців, без підзаголовків усередині секції, без emoji ✅/❌**

### Формат запису відкритих питань

- **`STAGES.md`** — перший стовпець таблиці `| Е |` = номер найраннішого етапу, який блокує рішення; таблиця сортується за цим стовпцем від меншого до більшого; якщо рішення блокує кілька етапів — ставити по найранішому, другорядний вказувати у колонці «Блокує» як `(також Ен)`
- **`MYKOLA.md`** — кожне питання починається з префікса `[Ен]`; список сортується за номером етапу від меншого до більшого
